第三方对接说明

接口用途：

我方平台生成用户登录凭证 ticket
第三方平台负责解密并校验
解密成功后，第三方可让用户直接登录
解密失败或校验失败时，第三方返回 401 或“登录失败”
1. 我方接口

请求方式：

GET /api/open/connect/session
请求头：

Authorization: Bearer <我方平台用户登录token>
说明：

该接口不需要传手机号
不需要传请求体
不需要传 query 参数
接口会根据当前登录用户自动生成登录凭证
成功返回示例：

{
  "code": 200,
  "message": "success",
  "result": {
    "ticket": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "expire_at": 1760000000,
    "expires_in": 600,
    "redirect_url": "https://www.zcat.cn"
  }
}
字段说明：

ticket：第三方需要解密的加密串
expire_at：票据过期时间戳，单位秒
expires_in：票据有效时长，当前为 600 秒
redirect_url：第三方平台入口地址
2. 加密说明

我方生成 ticket 时的明文格式：

uid|bid|timestamp
示例：

12345|40|1760000000
字段说明：

uid：我方平台用户 ID
bid：我方平台业务/租户 ID
timestamp：生成票据时的 Unix 时间戳，单位秒
3. 解密算法约定

加密算法：

AES-CBC
Padding：

PKCS5Padding
IV 规则：

IV = AES 密钥前 16 字节
密文编码：

hex
说明：

第三方收到 ticket 后，先按 hex 解码
再按 AES-CBC 解密
去掉 PKCS5Padding
最终得到明文：uid|bid|timestamp
4. AES 密钥

AES 密钥需要由我方线下单独提供给第三方。

说明：

该密钥不会通过接口返回
需要双方通过安全渠道单独同步
第三方必须使用与我方一致的密钥进行解密
5. 第三方校验规则建议

第三方解密成功后，建议至少校验以下内容：

明文是否能按 | 正常拆分成 3 段
uid 是否合法
bid 是否合法
timestamp 是否在有效期内
当前时间与 timestamp 的差值是否小于 600 秒
如果任一校验失败：

返回 401
或返回“登录失败”
如果校验通过：

允许用户进入第三方平台
并按第三方自身登录态体系完成登录
6. 第三方解密后示例处理逻辑

解密得到：

12345|40|1760000000
处理流程：

按 | 分割字符串
取出 uid=12345
取出 bid=40
取出 timestamp=1760000000
校验时间戳是否过期
校验 uid / bid 是否满足接入规则
校验通过后创建或绑定第三方登录态
7. 对接注意事项

ticket 是一次性时效凭证，建议尽快使用
当前有效期为 10 分钟
若超时，第三方应视为无效票据
若后续需要通过 URL 直接携带 ticket，我方可再补充跳转参数方案
